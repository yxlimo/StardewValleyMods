# 1.0 需求
## 综述
当前版本实现了对 i18n 的增量翻译，但是对其他的 json 文件和 xml 格式文件的选择性翻译支持还有问题，1.0 的需求就是为了解决这个问题

## 对于文件的处理方式
抽象出 fileOperator,对于不同的文件格式使用不同的库，json 使用 ts-jsonpath，tmx 文件使用 fast-xml-parser，但是对于外部调用者不需要关心具体的文件格式，查询以及更新的 query 对外部调用者应保持一致。

## 需要处理的问题
1. mod 下的 translation.json 合并到 config.json 中，只要不是 translationALl: true 的情况，都需要 translationKeys 去做指引
2. translationKeys 的语法尽量与 xpath 保持一致，但不需要严格一致，因为要处理更加复杂的情况，具体情况参考单独章节 

## translationKeys 字段需要实现的功能
- 针对路径中没有数组的情况，即 `{"A":{"B": "xxxx"}}`，query 与 xpath 格式保持一致，可以精确指向需要翻译的 key
- 如果路径中有数组，需要识别 Object 内部特征，不能用单纯的 `*` 或者 `index` 作为指引，会导致匹配结果错误（原作者可能会改变数组顺序）。
  - 参考 @../tests/resources/origin/TestMod/Data/dataWithArray.json。每一层数组都需要一个 location key 作为定位，可能是 key 本身的内容：`dreamy.kickitspot_jellishop`，也可能是 key 的 value：`{}"Id": "dreamy.kickitspot_Jelli"}`，需要设计相应的语法或者更复杂的 json 结构去匹配这样的特征。


## 目标
实现以上功能，并添加对应的 test case，跑通测试。
