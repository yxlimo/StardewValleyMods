# 0.2 版本需求

## 前置调研
Automate 中是否有类似 Group ID 的概念和相关 API ，可以快速判断某个箱子在不在某个 Group 中

## 需求内容
当前的 Hopper 只抓取上方箱子的物品进行匹配，现在需要变更为抓取上方箱子所在的 Automate Group 中的所有箱子的物品

### 原始版本
找到所有 Hopper 上方的 Chest，匹配并抓取物品
### 目标版本
#### 基础类型变更
新增 Class： AutomateChestGroup：包含被 Automate 连接在一起的一组箱子。有查找物品/删除物品/判断某个 Chest 是否在其中/重建 ChestGroup。 的方法，供 HopperIOGroup 调用。
AutomateChestGroup 入参为单个 chest 或 machine，自动遍历寻找所有的 Chest（查找 Automate 相关 api,能找到直接用，找不到就自行遍历）。注意：需要记录初始化的起点箱子，这是 HopperIOGroup 输入的起点。
HopperIOGroup 内 InputGroup 使用泛型約束，把单个 Chest Wrap 一个 class，实现和 HopperIOGroup 一样的方法。保证可以把 ChestWrap 和 ChestGroup 任意赋值给 HopperIOGroup.InputGroup。
当 GrabAutomateChestGroup 选项为 false, HopperIOGroup 的 InputGroup 为 ChestWrap，反之则为 ChestGroup。


#### 修改流程
1. 遍历所有 Hopper, 找到所有上方和下方均有 Chest 的 Hopper。
2. 针对上一步中的单个 Hopper，根据 GrabAutomateChestGroup 选项组成一个 HopperIOGroup，加入 Pipelines。
3. 当触发 ObjectChanged
   - 如果是 Hopper
     - 新增：组装一个新的 Pipeline，并加入 Pipelines
     - 移除：遍历 Pipelines，删除对应的 Pipeline
   - 如果是 Chest
     - 新增/移除：重建该 Location 下所有的 Pipelines（简化逻辑，不做精细检查）
4. 当触发 UpdateTicked, 遍历 Pipelines，处理每个 HopperIOGroup。（应该不用更改相关逻辑）

#### 5. Flood Fill 算法
```csharp
private List<Chest> FloodFillChests(Chest startChest, GameLocation location) {
    var visited = new HashSet<Vector2>();
    var result = new List<Chest>();
    var queue = new Queue<Vector2>();

    queue.Enqueue(startChest.TileLocation);
    visited.Add(startChest.TileLocation);

    while (queue.Count > 0) {
        var current = queue.Dequeue();
        // 添加到结果
        if (location.Objects.TryGetValue(current, out var obj) && obj is Chest chest) {
            result.Add(chest);
        }

        // 遍历四向邻居
        foreach (var neighbor in GetCardinalNeighbors(current)) {
            if (visited.Contains(neighbor)) continue;
            visited.Add(neighbor);

            // 检查是否是可连接的实体
            if (IsConnectableTile(neighbor, location)) {
                queue.Enqueue(neighbor);
            }
        }
    }

    return result;
}
```
