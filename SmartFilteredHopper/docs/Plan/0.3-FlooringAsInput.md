# 0.3 版本需求

## 综述
1. 在兼容 Automate Group 输入的基础上，让 Hopper 的输入不止是箱子，也可以是 Machine 和 Flooring。
2. Config 添加 Hopper 容量的选项，可以自定义 Hopper 容量，默认为36

## 修改方案
GrabAutomateChestGroup=true 的情况下
- Manager.findHopperConnector 在寻找 input 时要把 Flooring 和 Machine 当成合法 input 来构建，
- AutomateChestGroup 也要接受 Machine 和 Flooring 作为合法 input
  - 在这种情况下可能会出现 AutomateChestGroup 不存在 Chest 的情况，GetChests 和 GetItems 方法要兼容处理，外部调用者也要处理 GetItems，GetChests 为空数组的情况
- 参考 DeluxeGrabberFix 是如何处理 Automate 的多种地板连接件的( Automate 支持自定义地板种类作为连接件)，如何读取 Automate 配置的

修改容量的方法参考 BiggerAutoGrabber
