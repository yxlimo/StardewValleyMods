# SmartFilteredHopper 方案 B：Content Patcher 创建 Big Hopper

## 1. 核心思路

**不修改现有 hopper**，通过 Content Patcher 创建一个新的 BigCraftable：
- 使用 `SpecialChestType: BigChest` 获取游戏原生大箱子 UI（9×6=54格）
- 新物品作为现有 hopper 的"大容量升级版"

## 2. 关键限制

| 方式 | 可用容量 |
|------|----------|
| Content Patcher + BigChest | **54 格**（固定） |
| Harmony 补丁 | 可任意配置（方案 A） |

**Content Patcher 本身无法指定任意容量**，容量由 `SpecialChestType` 决定：
- `None` → 36 格
- `BigChest` → 54 格
- `MiniShippingBin` → 迷你发货箱

## 3. 实现

### 3.1 Content Patcher 配置

新建 `[CP] SmartFilteredHopper/content.json`：

```json
{
  "Format": "2.4.0",
  "Changes": [
    {
      "LogName": "Add Big Hopper",
      "Action": "EditData",
      "Target": "Data/BigCraftables",
      "Entries": {
        "SmartFilteredHopper.BigHopper": {
          "Name": "Big Hopper",
          "DisplayName": "{{i18n: big_hopper.name}}",
          "Description": "{{i18n: big_hopper.description}}",
          "Price": 0,
          "Texture": "TileSheets/Craftables",
          "SpriteIndex": 90,
          "Fragility": 0,
          "CustomFields": {
            "SmartFilteredHopper/IsHopper": "true",
            "furyx639.ExpandedStorage/SpecialChestType": "BigChest"
          }
        }
      }
    },
    {
      "LogName": "Add Big Hopper Recipe",
      "Action": "EditData",
      "Target": "Data/CraftingRecipes",
      "Entries": {
        "SmartFilteredHopper.BigHopper": {
          "Result": "SmartFilteredHopper.BigHopper",
          "Ingredients": [
            { "Object": 388, "Count": 30 },
            { "Object": 382, "Count": 10 }
          ],
          "IsMachine": false
        }
      }
    }
  ]
}
```

### 3.2 代码集成

```csharp
// Utill.cs - 识别 Big Hopper
public static bool IsBigHopper(StardewValley.Object obj) {
    return obj is Chest { SpecialChestType: Chest.SpecialChestTypes.BigChest }
        && obj.modData?.ContainsKey("SmartFilteredHopper/IsHopper") == true;
}

// 合并检测
public static bool IsHopper(StardewValley.Object obj) {
    return obj is Chest { SpecialChestType: Chest.SpecialChestTypes.AutoLoader };
}

public static bool IsOurHopper(StardewValley.Object obj) {
    return IsHopper(obj) || IsBigHopper(obj);
}
```

## 4. 方案对比

| 特性 | 方案 A（Harmony） | 方案 B（CP） |
|------|------------------|--------------|
| 容量 | 可任意配置 | **固定 54 格** |
| 实现方式 | Harmony 补丁 | Content Patcher |
| 冲突风险 | 中 | **极低** |
| 现有 hopper | 直接生效 | 需重新放置 |
| 滚动支持 | 需额外实现 | 原生无 |

## 5. 结论

方案 B 适合作为**补充方案**：
- 现有用户使用现有 hopper（方案 A 处理）
- 方案 B 提供一个无需 Harmony 的 54 格选项
