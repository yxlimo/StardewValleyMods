# SmartFilteredHopper 1.0: Hopper Output Limit

## 需求

1. **UI 以 Hopper 为单位**：每个 Hopper 有自己的限制配置
2. **打开方式**：在 Hopper 箱子界面的**右侧**放一个按钮，按按钮打开 UI 界面
3. **显示内容**：UI 界面显示过滤器物品图标，图标下方有输入框可输入数字
4. **限制值存储**：存在 Hopper 的 `modData` 中
5. **Transfer 校验**：如果输出箱中该物品数量 >= 设置值，跳过 transfer（在箱子满校验之前）

## 背景

### 现有配置项

```csharp
// ModConfig.cs
```

### 过滤器匹配逻辑

```csharp
// Manager.cs - shouldTransfer()
if (filterItem.QualifiedItemId != item.QualifiedItemId)
    continue;
// 只有当 CompareArtifactSource 开启且都是工匠物品时才比较来源
if (this.ctx.Config.CompareArtifactSource && Utill.HasPreserveSource(item) && Utill.HasPreserveSource(filterItem)) {
    string itemSource = Utill.GetPreserveTypeID(item);  // 如 "344" (果酱)
    string filterSource = Utill.GetPreserveTypeID(filterItem);
    if (itemSource != filterSource) continue;
}
// 只有当 CompareQuality 开启时才比较品质
if (this.ctx.Config.CompareQuality && filterItem.Quality != item.Quality)
    continue;
```

### 物品唯一标识策略

根据配置项不同，过滤器中的物品有以下几种唯一标识方式：

| 配置 | QualifiedItemId | Source ID | Quality | 完整 Key 示例 |
|------|-----------------|-----------|--------|--------------|
| 两者都关 | `O348` | - | - | `O348` |
| 仅 CompareArtifactSource | `O348` | `88` (葡萄) | - | `O348_88` |
| 两者都开 | `O348` | `88` | `2` (铱星) | `O348_88_2` |

### 风味来源物品的 Source ID

`Utill.GetFlavorSourceID()` 返回 `preservedParentSheetIndex.Value`，即原材料 ID：
- 果酱 (344): `613` (苹果), `304` (草莓) 等
- 酒 (348): `88` (葡萄), `396` (杏子) 等
- 果汁 (350): 同上
- 蜂蜜 (340): `876` (野花蜂蜜)

### 2. 限制值 Key 设计

```
SmartFilteredHopper/OutputLimit_{QualifiedItemId}[_{SourceId}][_{Quality}]
```

**示例**:
- `SmartFilteredHopper/OutputLimit_O348` - 普通葡萄酒，无限制
- `SmartFilteredHopper/OutputLimit_O348_88` - 葡萄酿的葡萄酒，SourceId=88
- `SmartFilteredHopper/OutputLimit_O348_88_2` - 铱星葡萄酿的葡萄酒

### 3. 动态 Key 构建方法

```csharp
/// <summary>
/// 根据当前配置构建 OutputLimit 的 modData key
/// </summary>
public static string BuildOutputLimitKey(Item filterItem, ModConfig config) {
    string key = $"SmartFilteredHopper/OutputLimit_{filterItem.QualifiedItemId}";

    if (config.CompareArtifactSource && Utill.HasPreserveSource(filterItem)) {
        string sourceId = Utill.GetFlavorSourceID(filterItem);
        if (!string.IsNullOrEmpty(sourceId)) {
            key += $"_{sourceId}";
        }
    }

    if (config.CompareQuality) {
        key += $"_{filterItem.Quality}";
    }

    return key;
}

/// <summary>
/// 从 modData key 解析出 QualifiedItemId（用于清理）
/// </summary>
public static string ParseQualifiedItemIdFromLimitKey(string key) {
    // key 格式: SmartFilteredHopper/OutputLimit_{QualifiedItemId}[_{SourceId}][_{Quality}]
    const string prefix = "SmartFilteredHopper/OutputLimit_";
    if (!key.StartsWith(prefix))
        return null;
    return key[prefix.Length..];
}
```

### 5. Hopper 物品变化的监听

当玩家往 hopper 里添加/移除过滤器物品时，需要重建 OutputLimit keys：

```csharp
// ModEntry.cs
helper.Events.Player.InventoryChanged += this.OnInventoryChanged;

private void OnInventoryChanged(object sender, InventoryChangedEventArgs e) {
    foreach (var item in e.Added) {
        if (item.Item is Chest chest && Utill.IsHopper(chest)) {
            OutputLimitHelper.RebuildOutputLimitKeys(chest, this.ctx.Config);
        }
    }
    foreach (var item in e.Removed) {
        if (item.Item is Chest chest && Utill.IsHopper(chest)) {
            OutputLimitHelper.CleanupInvalidLimitKeys(chest, this.ctx.Config);
        }
    }
}
```

## 详细实现

### 1. OutputLimitHelper.cs

```csharp
internal static class OutputLimitHelper {
    private const string KeyPrefix = "SmartFilteredHopper/OutputLimit_";

    /// <summary>
    /// 根据过滤器物品和配置构建 OutputLimit 的 modData key
    /// </summary>
    public static string BuildOutputLimitKey(Item filterItem, ModConfig config) {
        string key = $"{KeyPrefix}{filterItem.QualifiedItemId}";

        if (config.CompareArtifactSource && Utill.HasPreserveSource(filterItem)) {
            string sourceId = Utill.GetFlavorSourceID(filterItem);
            if (!string.IsNullOrEmpty(sourceId)) {
                key += $"_{sourceId}";
            }
        }

        if (config.CompareQuality) {
            key += $"_{filterItem.Quality}";
        }

        return key;
    }

    /// <summary>
    /// 重建 hopper 上所有 OutputLimit keys
    /// 根据当前过滤器物品和配置重新写入
    /// </summary>
    public static void RebuildOutputLimitKeys(Chest hopper, ModConfig config) {
        // 1. 收集当前有效的 keys
        var currentFilterKeys = hopper.Items
            .Where(i => i != null)
            .Select(i => BuildOutputLimitKey(i, config))
            .ToHashSet();

        // 2. 清理不在当前过滤器中的旧 keys
        var keysToRemove = hopper.modData.Keys
            .Where(k => k.StartsWith(KeyPrefix) && !currentFilterKeys.Contains(k))
            .ToList();
        foreach (var key in keysToRemove) {
            hopper.modData.Remove(key);
        }
    }

    /// <summary>
    /// 清理无效的 OutputLimit keys
    /// 当过滤器物品被移除时调用，删除已不存在的物品的 limit key
    /// </summary>
    public static void CleanupInvalidLimitKeys(Chest hopper, ModConfig config) {
        RebuildOutputLimitKeys(hopper, config);
    }

    public static int? GetOutputLimit(Chest hopper, Item filterItem, ModConfig config) {
        string key = BuildOutputLimitKey(filterItem, config);
        if (hopper.modData.TryGetValue(key, out string val) && int.TryParse(val, out int limit)) {
            return limit;
        }
        return null;
    }

    public static void SetOutputLimit(Chest hopper, Item filterItem, ModConfig config, int limit) {
        string key = BuildOutputLimitKey(filterItem, config);
        if (limit <= 0) {
            hopper.modData.Remove(key);
        } else {
            hopper.modData[key] = limit.ToString();
        }
    }
}
```

### 2. ChestOverlay.cs

```csharp
public class ChestOverlay : BaseOverlay {
    private readonly Chest chest;
    private readonly ItemGrabMenu rootMenu;
    private readonly List<SpriteButton> buttons = new();

    public ChestOverlay(ItemGrabMenu rootMenu, Chest chest) {
        this.rootMenu = rootMenu;
        this.chest = chest;

        // 按钮位置：基于 fillStacksButton
        var x = rootMenu.fillStacksButton.bounds.X + 80;
        var y = rootMenu.fillStacksButton.bounds.Y - 80;

        // Output Limit 按钮
        var limitButton = UIHelper.SideButton(x, y, "OutputLimit");
        limitButton.OnPress += OpenOutputLimitMenu;
        buttons.Add(limitButton);
    }

    public override void DrawUi(SpriteBatch spriteBatch) {
        foreach (var button in buttons) {
            button.Draw(spriteBatch);
        }
    }

    public override void ReceiveLeftClick(int x, int y) {
        foreach (var button in buttons) {
            if (button.Bounds.Contains(x, y)) {
                button.OnPress();
                break;
            }
        }
    }

    public override void ReceiveCursorHover(int x, int y) {
        foreach (var button in buttons) {
            button.IsHovered = button.Bounds.Contains(x, y);
        }
    }

    private void OpenOutputLimitMenu() {
        // 创建 Output Limit 菜单
        var menu = new OutputLimitMenu(this.chest);
        Game1.activeClickableMenu = menu;
    }
}
```

### 7. Rebuild 时重建 OutputLimit Keys

```csharp
public void RebuildIOGroups() {
    for (int i = this.IOGroups.Count - 1; i >= 0; i--) {
        var group = this.IOGroups[i];
        var (inputPos, output) = this.findHopperConnector(group.Hopper);
        if (inputPos == null || output == null) {
            group.Hopper.modData.Remove(modDataFlag);
            OutputLimitHelper.CleanupInvalidLimitKeys(group.Hopper, this.ctx.Config);
            this.IOGroups.RemoveAt(i);
            continue;
        }
        group.RebuildInput(this.ctx, inputPos.Value);
    }
}
```
